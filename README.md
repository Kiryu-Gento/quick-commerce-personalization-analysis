# SQL 분석 쿼리

이 폴더는 메인 분석(`../ecommerce_analysis.ipynb`, Pandas 기반)의 핵심 로직을 **MySQL로 재구현**한 결과물이다.
동일한 비즈니스 질문을 SQL로도 풀 수 있음을 보이며, 데이터웨어하우스 환경(직접 쿼리)을 가정해 작성했다.

## 쿼리 목록

### 01. 월별 인기상품 빈도지수 (Top 3)
**파일**: [`01_monthly_product_frequency_index.sql`](./01_monthly_product_frequency_index.sql)

**사용 기법**
- `AVG() OVER (PARTITION BY category)` — 카테고리별 월평균을 윈도우 함수로 계산
- `ROW_NUMBER() OVER (PARTITION BY 월 ORDER BY 빈도지수 DESC)` — 월별 순위

**비즈니스 의미**
빈도지수 = 해당 월 주문건수 / 카테고리 월평균. 1.0보다 크면 평소 대비 더 많이 팔린 달이고, 1.5 이상이면 시즌 상품으로 해석 가능. 월별 메인 배너·푸시 알림 노출 상품 선정 근거.

**원본 대비**: notebook Section 5의 Pandas `groupby + sort + head(3)` 로직을 윈도우 함수 두 개로 대체.

### 02. RFM 세그멘테이션
**파일**: [`02_rfm_segmentation.sql`](./02_rfm_segmentation.sql)

**사용 기법**
- `NTILE(5) OVER (ORDER BY ...)` — R/F/M 각각 5등분하여 점수화
- 다단계 CTE 체이닝
- `CASE WHEN`으로 점수 조합 → 표준 RFM 세그먼트 매핑 (Champion / Loyal / At Risk / Lost 등)

**비즈니스 의미**
원본 분석은 RFM 평균값만 비교했지만, 5점 만점 스코어링 + 세그먼트 분류까지 확장. 세그먼트별 차별화된 마케팅 전략 수립 가능.

**원본 대비**: notebook Section 6 확장 — 평균 비교 → 점수화 + 세그먼트 분류 + 권장 액션 매핑.

### 03. 휴면고객 위험군 분류
**파일**: [`03_dormant_customer_risk_groups.sql`](./03_dormant_customer_risk_groups.sql)

**사용 기법**
- `DATEDIFF` + `WHERE`로 휴면 정의(60일 이상 미구매)
- `NTILE(3)` — 빈도 기준 3등분 (High / Mid / Low)
- `CASE WHEN`으로 위험군별 권장 전략 매핑

**비즈니스 의미**
휴면고객을 단일 집단으로 보지 않고 복귀 가능성·잠재 가치 관점으로 세분화. 위험군별 차등 개입(쿠폰 금액·채널 차별화)으로 마케팅 ROI 극대화.

**원본 대비**: notebook Section 7의 `quantile(0.33)`/`quantile(0.66)` 수동 binning을 `NTILE(3)` 한 줄로 대체. SQL에서 분위수 기반 분할은 NTILE이 표준 패턴.

### 04. 코호트 리텐션 매트릭스 *(신규 분석)*
**파일**: [`04_cohort_retention.sql`](./04_cohort_retention.sql)

**사용 기법**
- 5단계 CTE 체이닝 (코호트 정의 → 활동 → 결합 → 집계)
- `DATE_FORMAT(MIN(주문일시), '%Y-%m')` — 코호트 정의(첫 구매월)
- `PERIOD_DIFF` — 코호트 기준 경과 개월수 계산
- 분모(코호트 사이즈) × 분자(활성 사용자) JOIN

**비즈니스 의미**
원본 분석의 휴면율 단일 수치(37.7%)를 **가입 시점별 시계열**로 입체화. 가로 방향은 같은 코호트의 시간 흐름에 따른 이탈을, 세로 방향은 가입 시기별 리텐션 차이(서비스 개선 효과)를 보여준다.

**원본 대비**: notebook에 없는 신규 분석. RFM(현재 상태)과 코호트(시간 흐름)는 이커머스 분석의 짝꿍.
